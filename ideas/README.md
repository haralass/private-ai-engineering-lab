# ideas/

Raw idea inbox before formal product specification.

## Directories

```
active/           Ideas currently being developed into product concepts
under-evaluation/ Ideas being researched and validated
future/           Good ideas not yet prioritized
rejected/         Ideas that didn't survive scrutiny — documented with reason
```

## From idea to product concept

1. Create a file in `active/` with the idea title and a paragraph description
2. Research: is there a clear problem? Who has it? What exists?
3. If the idea survives initial research, create a full `product-concepts/<name>/` directory using `templates/PRODUCT_IDEA_TEMPLATE.md`
4. Move the idea file to `under-evaluation/`

## Rejected ideas

Always document why an idea was rejected — not just "doesn't work" but the specific reason. The documentation prevents re-evaluating the same dead ends.
