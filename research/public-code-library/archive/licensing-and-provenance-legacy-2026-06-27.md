## License Risk Categories

### Green — Unrestricted Open Source
**MIT, Apache-2.0, BSD-2-Clause**

Can be used in any project, commercial or open-source. Apache-2.0 requires attribution in NOTICE file for substantial portions.

Repositories: 50+ of 68 retained

### Yellow — Verify Before Use

**NOASSERTION (kentcdodds.com, LeaVerou/stretchy)**
GitHub could not identify a recognized license. Could be a custom license, a README notice, or simply absent. Treat as "no license" until verified directly with the author.

**No License (pacocoursey/paco, paco/writer, raunofreiberg/interfaces, jh3y/vincent-van-git, maxboeck/mxb)**
No license file = all rights reserved by default under international copyright law. Code may be read but not reproduced, adapted, or distributed. Study only.

**Codrops (14islands/codrops-scroll-rig-tutorial)**
Free for personal/educational use; not free for commercial use. If a commercial project uses this pattern, re-implement independently without copying Codrops code.

### Red — Restricted

**FSL-1.1-ALv2 (saleor/storefront)**
Functional Source License: free for non-competing use. Cannot build a product that competes with Saleor/Vendo. After 4 years (or April 2029), converts to Apache-2.0. Read the FSL carefully before any code reuse.

**AGPL-3.0 (webstudio-is/webstudio)**
Copyleft extends to network use. Deploying a modified version of an AGPL project as a service (even privately) requires disclosing source. For research/study in isolation: fine. For integration into a proprietary product: prohibited without a commercial license.

**UNLICENSED (brunosimon/threejs-template-complex)**
Explicit all-rights-reserved declaration. No reuse of any kind. Use folio-2019 (MIT) instead.

**RESTRICTED (Shopify/dawn)**
Dawn license explicitly limits use to Shopify-hosted themes. If the project is a Shopify theme: permitted. If the project is anything else: prohibited.

## Provenance Notes

### Transferred Repositories
- `dip/cmdk` — originally `pacocoursey/cmdk`; transferred to `dip` org for maintenance continuity. Use `dip/cmdk` as the canonical source.
- `antfu-collective/vitesse` — originally `antfu/vitesse`; transferred to `antfu-collective` org. Use `antfu-collective/vitesse` as canonical.
- `epicweb-dev/epic-stack` — originally `kentcdodds/epic-stack`; transferred to `epicweb-dev` org when Kent created the EpicWeb organization. Use `epicweb-dev/epic-stack` as canonical.

### Fork Relationships
- `raunofreiberg/primitives` — fork of `radix-ui/primitives`. Not Rauno's original work; excluded.
- No other significant forks identified in retained repositories.

### Content vs. Code Split
Several repositories contain both MIT-licensed code and non-reusable personal content:
- Blog post text, images, personal photography — not covered by MIT
- Personal branding (colors, logo, name) — trademarked or reserved
- Portfolio project descriptions and case studies — personal work
- Design tokens that encode a specific visual identity — stylistically restricted even if MIT

### Attribution Requirements (Apache-2.0)
Apache-2.0 requires: (1) preservation of copyright notice, (2) preservation of license text, (3) NOTICE file for significant derivative works. Medplum repos (foomedical, chart-demo, demo-bots, hello-world) are all Apache-2.0.
